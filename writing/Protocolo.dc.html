Preparar Windows — WSL2 + DockerPrepare Windows — WSL2 + Docker

WSL significa Windows Subsystem for Linux. Es una forma de correr Linux dentro de Windows, sin máquinas virtuales pesadas ni borrar tu Windows. ¿Por qué Linux? Porque casi todo el mundo del desarrollo profesional vive ahí, y las herramientas funcionan mejor y más rápido.

WSL stands for Windows Subsystem for Linux. It's a way to run Linux inside Windows, without heavy virtual machines or erasing your Windows. Why Linux? Because almost the entire professional development world lives there, and the tools work better and faster.

1.1 — Instalar WSL21.1 — Install WSL2

  1. Hacé clic en el menú Inicio, escribí PowerShell.
  2. Click the Start menu and type PowerShell.
  3. Clic derecho sobre Windows PowerShell → Ejecutar como administrador.
  4. Right-click Windows PowerShell → Run as administrator.
  5. En la ventana azul, escribí este comando y apretá Enter:
  6. In the blue window, type this command and press Enter:
powershell 
wsl --install

Esto instala WSL2 y Ubuntu automáticamente. Cuando termine, reiniciá la computadora. Al volver a encender, se abre sola una ventana de Ubuntu que te pide crear un usuario y una contraseña de Linux. Elegí un nombre simple en minúsculas (ej: papa). La contraseña no se ve mientras la escribís (es normal). Anotala.

This installs WSL2 and Ubuntu automatically. When it finishes, restart the computer. On startup, an Ubuntu window opens on its own and asks you to create a Linux username and password. Pick a simple lowercase name (e.g. papa). The password stays invisible as you type (that's normal). Write it down.

★ LA REGLA DE OROTHE GOLDEN RULE

Pensar y decidir → Opus 4.8. Ejecutar y programar → Sonnet 4.6. Cuando la tarea VALE la pena (arquitectura, seguridad), Opus vale oro; cuando es escribir código siguiendo un plan, Sonnet te da velocidad sin perder calidad. No arranques con el más barato solo "para ahorrar".

Con Opus 4.8, Claude Code ya razona ALTO por defecto, así que no tenés que tocar nada para el trabajo normal. Para problemas difíciles, pedile explícitamente que piense más a fondo: agregá a tu pedido algo como "pensá esto con cuidado antes de tocar el código". With Opus 4.8, Claude Code already reasons at a HIGH level by default, so you don't need to touch anything for normal work. The words think hard or ultrathink are signals it recognizes to reason even deeper.

// ¿QUÉ ES EL TDD Y POR QUÉ IMPORTA?WHAT IS TDD AND WHY IT MATTERS

Test-Driven Development = "desarrollo guiado por pruebas": primero se escribe una prueba que define qué tiene que hacer el código, y RECIÉN DESPUÉS se escribe el código que la hace pasar. Te obliga a pensar QUÉ querés antes del cómo, y te deja una red de seguridad: si algo se rompe, las pruebas te avisan al instante. Es una de las mejores costumbres que podés agarrar desde el día uno.

Test-Driven Development means: first you write a test that defines what the code must do, and ONLY THEN you write the code that makes it pass. It forces you to think about WHAT you want before the how, and leaves you a safety net: if something breaks, the tests warn you instantly. It's one of the best habits you can pick up from day one.

Arquitectura LimpiaClean Architecture

Imaginá tu app como capas de una cebolla. En el centro está la lógica de tu negocio (lo que no cambia). Afuera están los detalles: la base de datos, la pantalla, internet. La regla: lo de afuera puede cambiar sin tocar el centro.

Picture your app as layers of an onion. At the center is your business logic (what doesn't change). Outside are the details: the database, the screen, the internet. The rule: the outside can change without touching the center.

Arquitectura HexagonalHexagonal Architecture

Tu lógica de negocio expone "puertos" (enchufes), y los "adaptadores" son los cables que conectan el mundo exterior a esos enchufes. Podés cambiar la pantalla o la base de datos sin tocar el corazón. Es lo que pediste en la Etapa 9.4.

Your business logic exposes "ports" (sockets), and "adapters" are the cables that connect the outside world to those sockets. You can swap the screen or the database without touching the heart. It's what you asked for in Stage 9.4.
